# Архитектура ИИ-Ассистента для Интервью в Реальном Времени

В данном документе описана концепция и варианты реализации технического решения для поддержки кандидата во время технических собеседований и лайвкодинга в режиме реального времени. 

---

## 1. Общий обзор системы

**Цель:** Перехват аудиопотока (голос интервьюера + голос кандидата), мгновенная обработка и получение подсказок от моделей ИИ на базе экосистемы Google Cloud с выводом ответов на второй монитор без задержек.

### Базовый сквозной процесс:
1. **Активация:** Пользователь зажимает горячую клавишу (Hotkey) на клавиатуре во время того, как интервьюер задает вопрос.
2. **Перехват звука:** Локальная утилита захватывает аудио из наушников (виртуальная петля) и микрофона, микшируя каналы в моно.
3. **Транскрибация (STT):** Аудиопоток мгновенно переводится в текстовый промпт.
4. **Оркестрация ИИ:** Текст отправляется на обработку в Google Cloud (двумя возможными путями).
5. **Вывод результата:** Сгенерированный ответ стримится на второй монитор в кастомный лаконичный интерфейс.

---

## 2. Варианты пайплайна передачи данных и получения ответа

Для утилизации различных грантов и кредитов Google Cloud (облачные $10 и целевые $1000 для GenAI App Builder) предусмотрены два сценария архитектуры.

### Вариант А: Прямой вызов Vertex AI API (Без собственного Data Store)
Этот вариант использует базовые языковые модели Gemini за счет стандартных облачных кредитов ($10 / месяц). Модель полагается исключительно на свои встроенные знания.

```
[ Текст вопроса ] ──► [ LiteLLM / Локальный прокси ] ──► [ Vertex AI API (Gemini) ] ──► [ Ответ из базы знаний модели ]
```

* **Как это устроена передача:**
    1. Локальный клиент отправляет текстовый запрос на локальный прокси-сервер (например, LiteLLM), имитирующий OpenAI API.
    2. Прокси-сервер подставляет сервисный JSON-ключ Google Cloud (`Vertex AI User`) и пересылает запрос в облако.
    3. Запрос обрабатывается моделью (например, `gemini-2.5-pro` или `gemini-3.5-flash`).
* **Плюсы:** Максимально низкая латентность (скорость ответа), простая настройка кода, гибкое управление контекстом через системный промпт.
* **Минусы:** Модель может галлюцинировать в специфических деталях; у неё нет доступа к вашим личным шпаргалкам и точным конспектам. Тратит базовые облачные доллары.

---

### Вариант Б: RAG через Vertex AI Agent Builder (С подключением Data Store)
Этот вариант задействует enterprise-платформу **Vertex AI Agent Builder** (ранее GenAI App Builder). Он позволяет ИИ искать информацию в ваших личных файлах (архитектурных схемах, книгах, базах вопросов и ответов) перед тем, как сгенерировать ответ. Этот вариант полностью покрывается грантом в **$1000**.

```
                   ┌────────────────────────────────────────────────────────┐
                   │               Vertex AI Agent Builder                  │
                   │                                                        │
[ Текст вопроса ] ─┼─► [ Векторный поиск ] ──► Проверка в Data Store        │
                   │                               │                        │
                   │                               ├─► (Есть) ─► Ответ по файлу ──► [ Итоговый ]
                   │                               │                        │  [  Ответ   ]
                   │                               └─► (Нет) ─► Фолбек на   ──► [         ]
                   │                                            знания Gemini       │
                   └────────────────────────────────────────────────────────┘
```

* **Как устроена передача:**
    1. Локальный скрипт отправляет распознанный текст через API к приложению (Search/Chat App) в Agent Builder.
    2. Платформа Agent Builder автоматически берет входящий текст и выполняет семантический поиск по вашему подключенному хранилищу документов (**Data Store**).
    3. **Сценарий 1 (Информация найдена):** Система извлекает точный кусок из вашей шпаргалки, передает его в Gemini как контекст, и модель формирует строго обоснованный ответ на основе вашего файла.
    4. **Сценарий 2 (Информации в Data Store нет):** Если релевантных документов не найдено, срабатывает встроенный механизм *fallback* (откат). Система перенаправляет запрос к общим параметрическим знаниям модели Gemini.
* **Плюсы:** Практически нулевой риск галлюцинаций по критическим темам, использование ваших личных лучших практик и заготовок кода, полная утилизация гранта в $1000.
* **Минусы:** Чуть более высокая латентность из-за промежуточного этапа поиска по документам (требуется оптимизация в виде стриминга токенов).

## 3. Сравнительная таблица пайплайнов

| Характеристика | Вариант А (Vertex AI Direct) | Вариант Б (Agent Builder + Data Store) |
| :--- | :--- | :--- |
| **Основной источник знаний** | Внутренние веса модели Gemini | Ваши файлы (PDF/MD) + Веса модели |

| **Способ интеграции** | LiteLLM прокси / Прямой SDK | API-запросы к Agent Builder / Voice Agent |

| **Сложность настройки** | Низкая (один конфигурационный YAML) | Средняя (требуется настроить Data Store и индекс) |

## 4. Рекомендации по оптимизации для минимизации задержек

Чтобы система на реальном интервью выдавала подсказки за 1-2 секунды, рекомендуется:
1. **Локальный STT:** Использовать локальный инференс *Whisper.cpp* или *Faster-Whisper* для моментального получения текста из аудио без отправки звуковых файлов в облако.
2. **Формат Data Store:** Загружать в Agent Builder не огромные книги, а атомарные Markdown-файлы, структурированные по принципу "Вопрос-Ответ" или емких технических тезисов.
3. **Стриминг:** Обязательно включить потоковую выдачу (`Stream: true`) в API, чтобы текст на втором мониторе появлялся посимвольно по мере генерации, исключая долгое ожидание.
